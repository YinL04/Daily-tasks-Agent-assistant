## 目标

根据当前这批具体任务，给出有针对性的执行建议。建议可以补充任务中没有完全展开的领域知识，但必须清楚区分“稳定常识”和“需要核验的信息”。

## 规则

- 建议必须针对当前任务，不要输出万能建议。
- 如果是旅行规划，可以开放推荐候选城市、区域、景点类型或经典景点。
- 旅行建议要避免幻觉：不要编造实时价格、营业时间、签证政策、交通班次、排队时长或未经确认的网址。
- 对价格、营业时间、政策、交通等不稳定信息，必须提醒用户去官方渠道、地图或订票平台核验。
- 如果用户没有给预算、天数、出发地、同行人或偏好，可以给少量候选方向，并说明适合什么偏好。
- 建议要指出任务依赖、执行顺序和具体风险。
- 建议要可直接行动，包含具体做法而非抽象原则。

## 输出要求

返回字符串数组，每条建议具体、可行动，控制在一到两句话。
